# W3M1 - Hadoop Single-Node Cluster on Docker

Docker 컨테이너 하나에 Hadoop 싱글 노드 클러스터(NameNode DataNode ResourceManager NodeManager)를 띄우고 HDFS에 디렉터리를 만들고 파일을 올리고 받아오는 것까지가 요구사항이었다.

Dockerfile과 core/hdfs/mapred/yarn-site.xml을 작성하고 entrypoint 스크립트가 컨테이너 시작 시 네임노드를 최초 1회만 포맷하고 이후엔 그대로 데몬을 띄우게 만들었다. Hadoop 기본 스크립트(start-dfs.sh 등)는 로컬호스트라도 SSH로 접속하는 구조라 싱글 노드에서도 SSH를 설치해서 정석대로 구성했다.

실제로 Docker Desktop을 켜서 빌드부터 HDFS 업로드와 다운로드까지 끝까지 돌려보다가 두 가지를 발견했다.

- ResourceManager 웹 UI(8088)가 호스트에서 접속이 안 됐다. yarn.resourcemanager.hostname이 localhost로 잡혀 있어서 컨테이너 내부 루프백에만 바인딩된 게 원인이었다. bind-host를 0.0.0.0으로 따로 지정해서 해결했다.
- Hadoop 다운로드가 archive.apache.org라 느렸는데 동료가 알려준 dlcdn.apache.org로 바꾸니 체감상 수십 배 빨라졌다.

요구사항 중 함정이라고 생각한 부분은 "container should be able to connect to the host machine's network" 문구였다. 문자 그대로 읽으면 host network 모드를 써야 할 것 같지만 이 모드는 Docker Desktop(Mac/Windows)에서는 제대로 지원되지 않는다. 그대로 적용하면 오히려 바로 다음 요구사항인 호스트에서 웹 UI 접속이 불가능해진다. 대신 포트 매핑(브리지 네트워크)으로 구현했고 실제 목적인 호스트에서 HDFS 접근 가능은 그대로 달성했다. 겉으로 보이는 요구사항 문구보다 실제 의도가 뭔지 생각해봐야 한다는 걸 느낀 지점이었다.
