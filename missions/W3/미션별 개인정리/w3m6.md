# W3M6 - Amazon Product Review using MapReduce

Amazon Reviews'23 데이터셋으로 상품별 리뷰 개수와 평균 평점을 같이 구하는 미션이었다. 로직은 M5와 거의 같은데 평균만이 아니라 리뷰 개수도 같이 출력해야 했다.

M5의 reducer를 그대로 재사용하다시피 했다. 어차피 합계와 개수를 계속 들고 있다가 마지막에 나누는 구조라 이미 계산은 되고 있던 개수를 출력문에 하나 더 끼워 넣는 것만으로 끝났다. 누적 로직을 짤 때 필요한 정보를 다 들고 있으면 나중에 출력 형식만 바뀌는 요구사항은 거의 공짜로 해결된다는 걸 체감했다.

이번엔 데이터가 CSV가 아니라 JSON Lines(한 줄에 리뷰 하나가 JSON 객체로) 형식이라 파싱 방식이 달랐다. 데이터셋은 전체 카테고리를 합치면 수백GB라 그중 가장 작은 카테고리인 Gift_Cards(15만 건)만 썼다.

제출 요구사항이 다른 M3~M5와 달랐다. "Submit all your Docker files"라는 문구가 명시돼 있어서 지금까지처럼 다른 미션의 클러스터를 참조만 하지 않고 이 미션 폴더 자체에 Dockerfile과 설정 파일을 통째로 복사해 넣었다. 요구사항 문구 하나가 미션마다 결과물 구조 자체를 바꿀 수 있다는 걸 확인한 사례였다.

"가장 많이 리뷰된 상품을 찾아라"라는 학습목표 문구가 있었다. 하지만 실제 기능과 프로그래밍 요구사항은 정렬이나 상위 N개 필터링을 요구하지 않고 전체 상품의 개수와 평균만 요구했다. README에 정렬해서 보는 명령어를 추가하는 정도로 취지를 살렸다. 검증은 출력의 리뷰 개수를 전부 더한 값이 원본 파일 줄 수와 정확히 일치하는 것으로 확인했다.
