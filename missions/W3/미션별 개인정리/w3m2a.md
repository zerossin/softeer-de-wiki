# W3M2a - Hadoop Multi-Node Cluster on Docker

M1을 멀티 노드로 확장하는 미션이었다. 마스터 노드 1개와 워커 노드 최소 1개 이상을 Docker 네트워크로 묶어서 하나의 HDFS/YARN 클러스터로 동작하게 만들고 MapReduce 잡까지 분산 처리로 돌려보는 게 목표였다.

M1과 가장 크게 다른 선택은 SSH를 없앤 것이다. M1은 Hadoop 기본 스크립트가 SSH로 데몬을 띄우는 방식을 그대로 썼는데 노드가 여러 개로 늘어나면 컨테이너끼리 키를 공유하고 known_hosts를 다뤄야 해서 복잡도만 늘어난다고 판단했다. 대신 각 컨테이너가 자기 역할(HADOOP_NODE 환경변수)에 맞는 데몬만 hdfs/yarn --daemon start로 직접 띄우고 워커는 마스터의 9000 포트가 열릴 때까지 기다렸다가 합류하게 만들었다. docker exec로 이미 컨테이너 안에 명령을 넣을 수 있는데 SSH를 또 구성하는 건 중복이라고 봤다.

다만 이건 요구사항이 강제한 게 아니라 순전히 내 설계 선택이었다. 이 미션 문구 자체가 Apache 공식 문서의 클러스터 구성 절차(포맷, 네임노드와 데이터노드 시작 등)를 거의 그대로 따르고 있다. 그래서 강사가 기대한 답은 오히려 SSH 기반 정석 방식이었을 가능성도 있다고 판단했다. 다만 이 부트캠프가 요구사항을 그대로 따르기보다 이유를 설명하는 걸 중시한다는 걸 감안하면 지금 방식이 방향에는 맞다고 보고 그대로 진행했다.

실제로 돌려보다가 mapreduce.application.classpath와 yarn.application.classpath를 명시하지 않으면 YarnChild 클래스를 못 찾아서 잡이 실패하는 문제를 겪었다. 두 값을 하드코딩해서 해결했다. wordcount와 pi 예제 모두 두 워커에 실제로 분산되어 처리되는 것까지 로그로 확인했다.
