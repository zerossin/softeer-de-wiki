# W3M5 - Average Rating of Movies using MapReduce

MovieLens 20M 데이터셋(평점 2천만 건)으로 영화별 평균 평점을 계산하는 미션이었다. 지금까지는 개수만 세면 됐는데 이번엔 평균을 구해야 해서 reducer가 값을 하나가 아니라 합계와 개수 두 개를 같이 들고 있어야 했다.

mapper는 movieId를 key로 평점을 rating,1(합계,개수) 형태로 emit한다. reducer는 같은 movieId끼리 합계와 개수를 계속 더하다가 마지막에 딱 한 번만 나눠서 평균을 낸다. 그룹별 평균을 다시 평균 내면(평균의 평균) 실제 값과 어긋나기 때문에 끝까지 합계와 개수로만 들고 다니다가 맨 마지막에 나누게 만든 게 이번 미션의 핵심이었다.

요구사항에 "efficiently process"라는 문구가 있길래 이번엔 Combiner도 추가했다. Combiner는 리듀서로 보내기 전에 각 Mapper 안에서 먼저 부분 합계를 내는 최적화다. 합계와 개수처럼 나중에 합쳐도 정확한 값에만 안전하게 쓸 수 있다. 실제로 2천만 건을 돌렸을 때 리듀서로 넘어가는 데이터가 2천만 건에서 14만 7천 건으로 줄어드는 걸 로그로 확인했다.

이 미션 요구사항 자체엔 특별한 함정은 없었다. 대신 mapper와 combiner와 reducer의 역할과 데이터 흐름에 대해 깊게 짚어봤다. 워커가 태스크를 어떻게 배정받는지도 짚었다. 워커가 스스로 일을 고르는 게 아니라 ResourceManager가 배정해준다는 것을 로그로 확인했다. 워커 2대가 있어도 실제로는 한쪽에만 몰릴 수 있다는 것과 매퍼 개수는 워커 개수가 아니라 데이터 크기로 정해진다는 것도 이때 이해했다.

검증은 전체 2천만 건으로 돌린 결과와 파이썬으로 직접 groupby해서 낸 평균이 소수점까지 정확히 일치하는 것으로 확인했다.
