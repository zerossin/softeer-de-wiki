# W3M4 - Twitter Sentiment Analysis using MapReduce

Sentiment140 데이터셋(트윗 160만 건)으로 감성 분석을 하는 미션이었다. 요구사항이 미리 정한 키워드 목록으로 긍정/부정/중립을 직접 분류하라고 명시하고 있어서 M3와 같은 Streaming 구조를 그대로 가져가되 mapper 안에 분류 로직을 새로 짰다.

트윗 텍스트를 단어 단위로 쪼개서 긍정 단어 집합과 부정 단어 집합에 각각 몇 개나 걸리는지 세고 많은 쪽으로 그 트윗 전체를 분류했다. 개수가 같으면(둘 다 0인 경우 포함) 중립으로 처리했다.

이 데이터셋은 사실 자체 라벨(0/2/4)이 있다. 확인해보니 학습용 160만 건짜리 파일엔 negative와 positive 라벨만 있고 neutral 라벨이 아예 없었다. 원래 라벨을 그대로 썼다면 애초에 3분류가 안 나왔을 거라 요구사항이 왜 굳이 키워드로 직접 분류하라고 했는지 납득이 갔다.

데이터셋(227MB)은 GitHub 용량 제한도 있고 이 저장소에 이미 있던 대용량 데이터셋 제외 관례를 따라서 커밋하지 않고 README에 다운로드 방법만 적었다.

이 미션을 하면서 mapper와 reducer가 각각 뭘 하는지에 대해 질문을 많이 주고받았다. worker와 mapper는 서로 다른 개념이라는 것도 이때 짚었다. mapper는 트윗 하나하나에 라벨을 붙이는 역할이고 reducer는 같은 라벨끼리 모인 걸 세서 합치는 역할이다. 최종적으로 저장되는 건 개별 트윗의 분류 결과가 아니라 카테고리별 합계 3줄뿐이라는 것도 이때 확인했다.
