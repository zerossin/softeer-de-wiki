# W3M3 - Word Count using MapReduce

200페이지 넘는 영어 전자책을 하나 골라서 파이썬으로 Mapper와 Reducer를 직접 짜서 워드카운트를 돌리는 미션이었다. 여기서부터는 지금까지 썼던 hadoop-mapreduce-examples 예제 jar를 안 쓰고 Hadoop Streaming으로 직접 짠 파이썬 스크립트를 붙이는 방식으로 넘어갔다.

브랜치는 M2b가 아니라 M2a에서 팠다. 이 미션이 필요한 건 그냥 돌아가는 클러스터뿐이고 M2b는 그 미션만의 결과물(설정 스크립트)이라 같이 가져올 이유가 없다고 판단했다.

전자책은 Project Gutenberg의 Pride and Prejudice를 썼다. mapper는 한 줄씩 받아서 단어를 뽑아 소문자로 바꾼 뒤 단어당 1을 emit한다. reducer는 Hadoop이 key(단어) 기준으로 정렬해서 넘겨주는 걸 이용해 같은 단어가 연속으로 오는 동안 개수를 누적하다가 바뀌면 출력하는 방식으로 짰다.

이번엔 요구사항과 실제 동작 사이에 특별한 함정은 못 찾았다. 오히려 "Python이면 JAR 대신 스크립트로 패키징"이라는 문구가 있어서 지금까지의 예제 jar 방식과 다르게 가야 한다는 게 명확했다.

로컬에서 파이프로 먼저 mapper와 reducer를 검증한 다음 실제 클러스터에 올려서 돌렸는데 결과가 완전히 똑같이 나오는 걸 확인했다. the/to/of/and 순서로 상위에 나오는 게 영어 소설로서 자연스러운 결과였다.
