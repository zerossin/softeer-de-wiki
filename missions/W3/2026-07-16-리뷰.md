# 2026-07-16 리뷰

오전에는 지난 과제의 코드 리뷰를 받았다. Docker Compose 사용 여부, NameNode 유저 설정, hadoop tmp directory 설정 등에서 요구사항을 문자 그대로 따르기보다는 현업의 구조를 이해하고 타당성을 검토하는 것이 중요하다는 점을 배웠다.

같은 시간에 수업에서는 데이터 처리 파이프라인에서 Classification을 어느 단계에서 할지 선택하는 것의 중요성, 캐시를 통한 중복 처리 제거, 프로토타입의 개념을 다루었다. 프로토타입은 먼저 가치가 있는지 검증하는 단계이고, 10억 개 규모는 그 이후에 안정성을 고려해서 다시 논의해야 한다는 점을 이해했다. 또한 좋은 비즈니스 아이디어와 좋은 데이터 엔지니어링 아이디어는 다르다는 점, 이터레이션의 중요성, 자신이 모르는 것을 표현하는 메타인지의 가치를 배웠다.

오후에는 팀의 W2 과제 프로토타입에 대한 피드백을 받았다. 주장에 근거가 부족했고, 샘플링으로 먼저 검증하지 않았으며, 기존 방식과 비교하거나 사후 검증을 하지 않았다는 지적을 받았다. 또한 개인 컴에서 먼저 검증한 뒤 클러스터로 확장하는 방식이 더 효율적이었을 것이라는 점과 비용을 고려해야 한다는 조언을 받았다.

저녁에는 10시까지 남아 DA 프로젝트의 모델링을 이어갔다. 칠판에 설계를 그리며 적극적으로 의문을 제기하고 의견을 냈고, 팀원들과 함께 토의하면서 지식을 체화시켜나갔다.
