# IMF 홈페이지에서 직접 데이터를 가져오는 방법

위키피디아 웹 스크래핑 대신 IMF에서 제공하는 공식 REST API 등을 활용하는 방법을 생각해볼 수 있다. HTML 구조 변경의 영향을 받지 않아 데이터 수집의 안정성과 신뢰성을 높일 수 있을 것으로 보인다. BeautifulSoup 대신 requests를 이용해 JSON 데이터를 직접 수집하도록 ETL의 Extract 단계를 변경한다.

# 데이터 갱신 시 ETL 프로세스 변경 방안

데이터를 덮어쓰는 것은 좋지 않아 보인다. 경영진에게는 시간의 흐름이 더 중요할 수 있기 때문이다. 따라서 발표 시점을 나타내는 컬럼(Release_Version와 같은)을 추가하여 스냅샷 방식으로 누적 저장하는 것이 좋아보인다. Load 단계에서 replace 대신 append를 사용하고, 최신 데이터 조회 시에는 가장 최근 버전만 조회하며 필요에 따라 과거 버전과 비교 분석할 수 있도록 ETL과 SQL을 구성한다.
