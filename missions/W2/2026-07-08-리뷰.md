# 2026-07-08 리뷰

오전과 오후 초반에는 DA 수업을 들었다. 이번 수업에서는 식별자, 정규화, 물리 모델링을 중심으로 데이터베이스 설계에 대해 학습했다. MySQL, Oracle, PostgreSQL 등 DBMS별 저장 구조와 인덱스 방식의 차이, 자연키와 대리키, UUID, 정규화와 반정규화, 물리 모델링 시 고려해야 할 요소 등을 학습하며 논리 모델을 실제 데이터베이스로 구현하는 과정에 대해 이해할 수 있었다. 또한 현업에서의 의사결정 방식과 실무 경험을 함께 들으며 기술적인 내용뿐 아니라 실무적인 관점도 접할 수 있었다.

이후에는 AWS와 Docker를 활용하여 JupyterLab 환경을 EC2에 배포하는 과제를 수행했다. AWS 계정을 생성하고 EC2 인스턴스를 구성한 뒤, Docker 이미지를 생성하여 ECR에 업로드하고 EC2에서 이미지를 내려받아 실행하는 전체 배포 과정을 진행하였다. 또한 User Data를 활용한 Docker 자동 설치와 포트 설정, 원격 접속 등을 경험하며 실제 배포 과정을 수행하였다.

과제를 진행하는 과정에서는 여러 시행착오를 겪었다. Apple Silicon과 EC2 간 아키텍처 차이로 인해 이미지를 다시 빌드해야 했고, 공용 AWS 계정의 임시 인증 토큰(MFA)과 권한 문제, 로컬과 원격 환경을 오가며 작업하는 과정에서의 혼동, 보안 정책으로 인해 SSH 접속이 차단되는 상황 등을 경험하였다. 문제를 하나씩 해결하며 배포를 완료할 수 있었고, 최종적으로 EC2에서 JupyterLab을 정상적으로 실행하는 것을 확인하였다.

추가 요구사항을 수행하는 과정에서는 요구사항을 잘못 확인하여 W2 노트북이 아닌 W1 노트북을 배포하였다. 이후 요구사항을 다시 확인한 뒤 이미지를 수정하여 재배포를 진행하였다. 또한 작업에 집중하다 보니 팀원들과 진행 상황을 충분히 공유하지 못했고, 과제를 마친 뒤에는 피로로 인해 당일 회고를 작성하지 못하고 다음 날 작성하게 되었다.
