# 2026-07-07 리뷰

오전에는 W2M5 과제를 수행했다. 예제를 구현하는 과정에서 데이터에 `u`, `im`과 같이 실제 분석에는 불필요해 보이는 데이터가 포함되어 있는 것을 확인했다. 우선 요구사항에 맞게 구현을 완료했으며, 실제 프로젝트에서는 이러한 데이터를 제거하거나 정제하는 과정이 필요할 수 있겠다는 생각을 하게 되었다.

이후에는 팀 프로젝트 주제를 기획하고 논의하는 시간을 가졌다. 강사님의 피드백을 바탕으로 분석 대상과 고객을 구체화하고, 실제 비즈니스적 가치를 가질 수 있는 프로젝트를 만드는 방향으로 아이디어를 고민하였다.

먼저 월드컵 대표팀 선수별 광고 모델 리스크를 분석하는 프로젝트를 기획하여 팀에 제안하였다. 그러나 팀 논의 과정에서 플레이스토어 리뷰 데이터를 활용하는 방향으로 의견이 모였고, 이에 따라 기존 아이디어를 발전시켜 카카오톡 UI 개편 이후의 리뷰 변화를 분석하는 프로젝트를 다시 기획하여 제안하였다. 이후 해당 방향을 중심으로 프로젝트를 구체화하게 되었다.
프로젝트를 준비하는 과정에서는 데이터 수집 방식도 검토하였다. 처음에는 `google-play-scraper` 라이브러리를 활용하여 데이터를 수집했지만, 과제 요구사항에 따라 라이브러리의 동작 방식을 분석한 뒤 `urllib`를 이용해 동일한 기능을 직접 구현하였다. 이후 팀에서는 Selenium을 이용해 일부 리뷰를 수집하는 방향을 논의하였고, 데이터 수집 범위와 방법에 대해서도 의견을 나누었다.
